---
name: no-arch-changes-without-discussion
description: 不要擅自改动架构，必须先讨论确认后再修改
metadata:
  type: feedback
---

**规则**：永远不要在没有和用户讨论并确认的情况下，擅自改动项目架构。

**Why**：用户明确表示不希望我擅自改动架构。架构变更可能带来意外的副作用，影响项目稳定性，需要用户主动评估风险。

**How to apply**：
- 当任务涉及架构调整（删除/新增模块、重构目录结构、替换技术方案、删除已有功能模块等），先停下来问用户"是否需要这样做"
- 如果我认为有必要修改，提供修改方案（为什么改、怎么改、影响范围），等待用户确认
- 在修改相关文档后再动手改代码
- 小范围的 bug 修复、代码补全、UI 微调不需要确认
