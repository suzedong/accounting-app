# 界面设计

## 1. 设计原则

- **保留现有 UI 风格**：现有 HTML/CSS 已有完整的视觉设计，重构期间不改变视觉风格
- **桌面优化**：固定窗口尺寸，支持窗口拖拽、最小化/最大化
- **离线状态展示**：同步状态可视化
- **减少层级**：桌面应用可承载更多功能在同一页面，减少页面跳转

## 2. 页面结构

## 4. 新增 UI 元素

### 4.1 同步状态指示器

在导航栏或页脚显示同步状态：

```
┌──────────────────────────────────────────────┐
│ 首页 记录 预算 统计 差旅    [🔄 同步] ● 已同步 │
└──────────────────────────────────────────────┘

状态：
● 已同步     - 绿色圆点
↻ 同步中     - 旋转图标
○ 未同步     - 灰色圆点（无连接）
⚠ 有冲突     - 黄色警告
```

### 4.2 抽屉标题栏

调试抽屉的标题栏采用扁平白色设计：

- 纯白背景（非渐变）
- 底部细线分隔（`border-bottom: 1px solid #e0e0e0`）
- 深色标题文字
- 灰色图标按钮，hover 时变蓝
- 关闭按钮使用 `Close` 图标（Element Plus Icons）

### 4.3 Skill 标签

Skill 标签位于消息气泡**上方**（非气泡内部）：

```
┌─────────────────────────┐
│ [create_record]  95%     │  ← skill tag（灰边框 pill，在气泡外）
│                          │
│ 已为您记录：              │
│   餐饮 支出 ¥35          │  ← message bubble
│   个人账户 2026-05-26    │
│                          │
│ [确认] [编辑] [取消]      │
└─────────────────────────┘
```

样式规范：
- `.message` 使用 `flex-direction: column`
- `.message.ai` 使用 `align-items: flex-start`
- skill tag 为半透明 pill（灰边框 `#e0e0e0`，无渐变，圆角 10px）
- 位置：`margin-left: 4px; margin-bottom: 4px`

### 4.4 思考过程展示

思考过程始终可见，折叠后不消失：

- 3 步流程：`intent`（意图识别）→ `execute`（执行入库）→ `done`（完成）
- 通过 `thinkingStep` ref 动态绑定步骤状态
- 折叠时：`thinkingStep` 设为 `'done'`，气泡保持可见，透明度降至 0.7
- 不隐藏：`collapseThinking` 不再设置 `thinkingVisible.value = false`
- 折叠后的思考过程作为历史记录保留在消息流中

### 4.5 图片粘贴（剪贴板）

ChatInput 支持剪贴板图片粘贴（Cmd+V / Ctrl+V）：

- `@paste` 事件监听器检测剪贴板中的图片
- 自动读取为 File 对象，走与文件选择相同的图片加载流程
- 粘贴后清空剪贴板，防止重复粘贴
- 图片预览在输入框上方显示

## 5. 现有页面改动说明

### 5.1 首页（AI 对话）

**布局不变**，改动仅在数据获取层：

- 记录列表从 `fetch('/api/records')` 改为 `invoke('get_records')`
- 对话历史从 localStorage 改为 SQLite
- 图片 OCR 从 `fetch('/api/ai/ocr')` 改为 `invoke('ocr_recognize')`
- AI 调用从 `fetch('/api/ai/dispatch')` 改为前端直连百炼 API

**新增元素**：同步状态指示器

### 5.2 记录管理页

**布局不变**，改动在数据源：

- 所有 CRUD 操作从 NocoBase API 改为 SQLite
- 筛选和分页由 SQL 查询完成（不再前端计算）
- 分页器保留，SQL OFFSET/LIMIT 实现

### 5.3 预算页

**布局不变**：

- 预算数据从 SQLite 读取
- 统计计算由 SQL GROUP BY 完成

### 5.4 统计页

**布局不变**：

- 数据获取从 `pageSize=10000` 改为 SQL 聚合查询
- ECharts 渲染（替换了原 Vue Data UI）

### 5.5 差旅补助页

**布局不变**：

- 数据从 SQLite 读取
- 补助计算逻辑不变
